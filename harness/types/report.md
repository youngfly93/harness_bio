# 报告生成 Harness

> 历史数据：25 个 session。
> 核心问题：数值与源数据不一致（最高频）、python-docx 行号偏移崩溃。

---

## 工具选择

| 工具 | 适用场景 |
|------|---------|
| python-docx | 复杂格式控制、模板化批量生成 |
| R officer | 与 R 分析流程一体化 |
| R rmarkdown → docx | 简单报告、图表自动嵌入 |

---

## 不可违反的规则

### 数值嵌入（P0）
- 所有数值 **必须** 从源数据文件程序化读取
- 禁止在报告文本中硬编码任何数字
- 实现方式：先生成数值摘要 JSON/CSV/TSV → 报告模板从中读取
- 推荐同时保存：
  - `numeric_reference.tsv`：源数据标准数值
  - `report_claims.tsv`：报告中实际写入的数值

### 图片嵌入
- 嵌入前检查图片路径存在且文件大小 > 0
- 嵌入后检查 docx 中图片数量与预期一致
- 图片尺寸统一（建议宽度 15cm，等比例缩放）

### 编辑方式（P1 — 历史高频 bug）
- **使用段落内容匹配**，不使用行号定位
- python-docx 的 XML 重排会导致行号偏移 → 按行号编辑必崩
- 查找目标段落 → 替换内容，而非 `paragraphs[line_number]`

### 字体规范
- 中文正文：黑体 / 宋体
- 英文 / 数字：Times New Roman
- 表格 / 代码：等宽字体
- 标题层级与模板一致

### 交叉引用
- 图表编号与正文引用 **必须** 一致
- 如增减图表，全文更新引用编号

## 交付前验证

```
□ 所有图片正常显示（非空白占位符）

□ 页眉页脚不含 AI 工具名称

□ 中英文字体正确
□ AI 痕迹扫描通过（→ harness/delivery/ai_scan.sh）
```

## 已知陷阱

| 陷阱 | 防护措施 |
|------|---------|
| 硬编码数值导致报告与源数据不一致 | 程序化读取，禁止手动 |
| python-docx 行号偏移崩溃 | 段落内容匹配，非行号 |
| 图片路径错误显示空白 | 嵌入前验证路径和文件大小 |
| figure legend 描述与实际图不符 | legend 从图表元数据自动生成 |
| 增删图表后编号不连续 | 全文编号刷新检查 |
