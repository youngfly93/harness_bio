# 多组学整合分析 Harness

> 适用于：16S + 代谢组、蛋白组 + 转录组、蛋白组 + PTM 等联合分析。

---

## 标准流程

```
1. 各组学独立分析（参照对应的单组学 Harness）
2. 样本 ID 匹配验证（第一步，不可跳过）
3. 数据标准化与对齐
4. 相关性分析（Mantel / Procrustes / MOFA / mixOmics）
5. 联合通路分析
6. 整合可视化（联合热图、桑基图、网络图）
```

---

## 关键检查点

### 样本 ID 匹配（P0 — 不可跳过的第一步）
- 各组学数据的样本 ID **必须完全匹配**
- 匹配前输出交集/差集报告
- 有样本不匹配时 → 停下来报告，不自行丢弃

### 归一化方法
- 各组学使用各自合适的归一化方法，**不能混用**
- 16S: rarefaction / CSS / CLR
- 代谢组: log2 + Pareto scaling
- 蛋白组: MaxLFQ / TMT ratio normalization
- 转录组: DESeq2 vst / TPM（根据下游分析选择）

### 整合分析
- 相关性分析的多重检验校正（Mantel test 的置换检验次数 ≥ 999）
- MOFA 的因子数量选择需要有依据（variance explained）
- 整合结果的生物学解释 → 如有争议，标记待人工确认

---

## 已知陷阱

| 陷阱 | 防护措施 |
|------|---------|
| 样本 ID 格式不一致（S1 vs Sample_1） | 整合前做 ID 标准化映射 |
| 不同组学的归一化方法混用 | 在各组学独立分析中固定方法 |
| Mantel test 置换次数过少 | 设置 permutations ≥ 999 |
| 整合可视化中样本顺序不对齐 | 统一按 metadata 排序 |
| 单组学结果未独立验证就做整合 | 各单组学先独立通过审计 |
